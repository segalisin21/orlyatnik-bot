# Инструкция: админ-панель бота «Орлятник 21+»

Полное руководство по работе менеджера/администратора в Telegram-боте.

---

## 1. Кто считается админом

Админом считается Telegram-аккаунт, чей **numeric user id** указан в переменной окружения **`ADMIN_CHAT_ID`** на Railway (или в `.env` локально).

- **Один админ:** `ADMIN_CHAT_ID=123456789`
- **Несколько админов:** через запятую — `ADMIN_CHAT_ID=123456789,987654321`

### Как узнать свой id

1. Напишите боту любое сообщение.
2. Откройте в браузере:  
   `https://api.telegram.org/bot<ВАШ_BOT_TOKEN>/getUpdates`
3. Найдите `"from":{"id":123456789}` — это ваш id.

**Обязательно:** каждый админ должен хотя бы раз написать боту **`/start`**, иначе Telegram не даст боту присылать вам сообщения (уведомления о чеках не дойдут).

---

## 2. Как открыть админ-меню

От аккаунта админа в личке с ботом отправьте:

| Команда | Что происходит |
|---------|----------------|
| **`/start`** | Открывается **админ-меню** (не обычное меню участника) |
| **`/admin`** | То же самое — админ-меню |

Появится сообщение:

> *Привет! Админ-меню. Уведомления о чеках приходят сюда — подтверждай кнопкой под сообщением.*

И три кнопки:

```
📢 Рассылка    📊 Статистика
⚙ Настройки
```

## 3. Главная задача админа — подтверждение оплаты

### Как приходит уведомление

Когда участник присылает **чек** (фото или документ) в статусе ожидания оплаты, бот:

1. Меняет статус участника на **`PAYMENT_SENT`**.
2. Отправляет **всем админам** сообщение с:
   - мероприятием (Орлятник / Пижамник);
   - `@username` и `user_id`;
   - пометкой о второй брони (если есть);
   - **полной анкетой**;
   - **фото/файлом чека**;
   - кнопкой **`✅ Подтвердить оплату`**.

Если подключена **ЮKassa**, аналогичное уведомление приходит после успешной оплаты (тоже с кнопкой подтверждения).

### Как подтвердить (рекомендуемый способ)

1. Проверьте чек и анкету.
2. Нажмите **`✅ Подтвердить оплату`** под сообщением.

Бот автоматически:

- ставит статус **`CONFIRMED`** в Google Sheets;
- проставляет `final_sent_at`;
- отправляет участнику **финальную цепочку** (для Орлятника):
  - картинка-поздравление (если задан URL в настройках);
  - текст «Ты в списке!» + задаток 8 000 ₽;
  - картинка чек-листа вещей;
  - кнопки: **идеи образов**, **Эльвира**, **Кристина**, **Ещё одна путёвка**;
- дописывает к вашему сообщению **`✅ Подтверждено`** и убирает кнопку.

Если кнопка не сработала («Уже подтверждено или участник не найден») — оплату уже подтвердили или статус в таблице другой.

### Альтернатива — через Google Sheets

Можно вручную поставить в листе **«Участники»** статус **`CONFIRMED`**.

- Если **`final_sent_at` пустой** — бот **сам** отправит финальное сообщение участнику в течение **~2 минут** (фоновый cron).
- Если **`final_sent_at` уже заполнен** — повторно финал не уйдёт.

Кнопка в Telegram удобнее: финал уходит **сразу**, без ожидания cron.

## 4. Статистика

**Путь:** Админ-меню → **📊 Статистика**

Показывает число участников **с заполненным `chat_id`** в Google Sheets:

| Показатель | Кто попадает |
|------------|--------------|
| **Всего с chat_id** | Все, кто хоть раз писал боту |
| **Подтверждённые** | Статус `CONFIRMED` |
| **Ждут оплаты / чек** | Статусы `WAIT_PAYMENT` и `PAYMENT_SENT` |

Учитываются оба листа: **«Участники»** и **«Пижамник»** (если есть).

---

## 5. Рассылка

**Путь:** Админ-меню → **📢 Рассылка**

### Шаг 1 — выбор аудитории

| Кнопка | Кому уйдёт |
|--------|------------|
| **Всем в таблице** | Все с `chat_id` (любой статус) |
| **Подтверждённые** | Только `CONFIRMED` |
| **Ждут оплаты / чек** | `WAIT_PAYMENT` + `PAYMENT_SENT` |

### Шаг 2 — текст

Бот попросит:

> *Напиши текст сообщения для рассылки (одним сообщением). Отправь /cancel чтобы отменить.*

Отправьте **одним сообщением** готовый текст рассылки.

### Шаг 3 — результат

Бот ответит, например:

> *Рассылка завершена. Отправлено: 47, ошибок: 2.*

Ошибки бывают, если пользователь заблокировал бота или не нажимал `/start`.

### Отмена

На любом шаге (ожидание текста рассылки или нового значения настройки) отправьте **`/cancel`**.

## 6. Настройки (редактирование текстов и цен без деплоя)

**Путь:** Админ-меню → **⚙ Настройки** → **Орлятник 21+**

(Пижамник — только если включён `FEATURE_PIZHAMNIK_UI_ENABLED`; сейчас в проде обычно только Орлятник.)

### Как менять параметр

1. Выберите нужную кнопку (например, **«Задаток (₽)»**).
2. Бот покажет: *Введите новое значение… /cancel — отмена.*
3. Отправьте **одним сообщением** новое значение.
4. Бот ответит: *✅ Сохранено… значение записано в лист «Настройки» — бот уже использует его.*

Изменения сразу попадают в Google Sheets и в память бота — **перезапуск не нужен**.

### Что можно редактировать (Орлятник)

#### Даты и смены

- Даты заезда/выезда
- Ближайшая смена (даты)
- Смена по умолчанию
- Список смен (через запятую)

#### Тексты меню и воронки

- Приветствие после выбора «Орлятник 21+»
- Текст программы (если нет фото)
- Текст условий и стоимости
- Текст согласия на обработку ПД
- Вопросы анкеты (ФИО, город, дата рождения, с кем едет, телефон, смена)
- Инструкция после оплаты

#### Отзывы

- Подпись кнопки отзывов
- Текст перед ссылкой на отзывы
- Ссылка на пост с отзывами

#### Картинки (URL)

- Обложка программы (`PROGRAM_COVER_PHOTO`)
- Фото программы 1–3
- Инфографика: цены (`CONDITIONS_PRICE_PHOTO`)
- Инфографика: условия (`CONDITIONS_TERMS_PHOTO`)
- Поздравление после оплаты
- Чек-лист вещей
- Ссылка на идеи образов (Pinterest/ВК)

#### Цены и оплата

- Цена (₽) — справочное поле в настройках
- **Задаток (₽)** — важно держать **8000**
- Реквизиты Сбер

#### Контент для LLM и ответов

- Локация
- Что входит
- Что взять с собой
- Ответы на возражения (дорого, один, не пью, нет компании)
- Ссылка на фото/видео канала
- Контакты менеджеров (текст)
- Ссылки на личку Эльвиры и Кристины

#### После подтверждения

- Текст после подтверждения оплаты
- Кнопки второй путёвки (тексты «для себя / для другого» и т.д.)

**Важно про URL картинок:** нужны **прямые HTTPS-ссылки**, которые Telegram может скачать (Google Drive «share» часто не работает — лучше залить на хостинг/Telegram-канал и взять прямую ссылку).

### Настройки Пижамника

Если включён UI Пижамника: **⚙ Настройки → Пижамник** — отдельный лист **«Настройки Пижамник»** с ключами: цена, задаток, остаток, лимит мест, тексты программы/условий, сообщения после чека, возврат и т.д.

| Команда | Назначение |
|---------|------------|
| **`/cancel`** | Отменить текущее действие (рассылку или ввод настройки) |
| **`/diag`** | Проверка связи с OpenAI: `LLM OK: …` или `LLM ОШИБКА: …` |
| **`/start`**, **`/admin`** | Вернуться в админ-меню |

## 8. Google Sheets — что смотреть вручную

Основная CRM — таблица Google Sheets (`GOOGLE_SHEET_ID`).

| Лист | Назначение |
|------|------------|
| **Участники** | Орлятник + новые пользователи |
| **Пижамник** | Участники Пижамника |
| **Настройки** | Тексты/цены Орлятника (дублирует админку) |
| **Настройки Пижамник** | Тексты/цены Пижамника |
| **Ответы** | Кэш готовых ответов LLM |
| **Логи** (если есть) | Переписка |

### Ключевые колонки участника

| Колонка | Назначение |
|---------|------------|
| `status` | Этап воронки: `NEW`, `INFO`, `FORM_FILLING`, `FORM_CONFIRM`, `WAIT_PAYMENT`, `PAYMENT_SENT`, `CONFIRMED`, `WAITLIST` |
| `fio`, `city`, `dob`, `phone`, `shift`, `companions`, `comment` | Анкета |
| `payment_proof_file_id` | ID чека в Telegram |
| `final_sent_at` | Финальное сообщение уже отправлено (ISO-дата) |
| `booking_ref`, `booking_scope` | Вторая путёвка (`self` / `other`) |
| `event` | `orlyatnik` / `pizhamnik` |
| `chat_id` | Нужен для рассылок и сообщений от бота |

Ручное редактирование статуса в таблице — запасной способ управления, если кнопка в боте недоступна.

## 9. Автоматика, о которой нужно знать

| Что | Когда |
|-----|--------|
| **Финал после CONFIRMED** | Каждые **2 минуты**, если статус `CONFIRMED` и `final_sent_at` пуст |
| **Напоминания «давно не писал»** | **Ежедневно в 10:00 МСК** (не чаще 1 раза в 7 дней на человека) |
| **Напоминание об остатке (Пижамник)** | 11 марта в 10:00 МСК |

## 10. Типовые сценарии

### Участник прислал чек — что делать?

1. Дождаться сообщения от бота с анкетой и чеком.
2. Сверить сумму (задаток **8 000 ₽**) и ФИО/смену.
3. Нажать **✅ Подтвердить оплату**.
4. Убедиться, что участник получил поздравление и чек-лист.

### Нужно срочно поменять цену или текст условий

1. **⚙ Настройки** → нужный параметр → новое значение.
2. Или правка напрямую в листе **«Настройки»** (колонки A = ключ, B = значение).  
   После правки в таблице без бота — перезапуск сервиса подтянет конфиг при старте; через админку — сразу.

### Нужно написать всем подтверждённым

**📢 Рассылка** → **Подтверждённые** → текст.

### Бот «не отвечает умно» на вопросы

1. **`/diag`** — проверить OpenAI.
2. Проверить **`OPENAI_API_KEY`** на Railway.
3. Проверить лист **«Настройки»** — там должны быть актуальные факты (цены, задаток 8000, локация).

### Вторая путёвка на Орлятник

После `CONFIRMED` участник может нажать **«Ещё одна путёвка»** или написать «Хочу забронировать». В CRM создаётся **новая строка**; в уведомлении админу будет пометка `2-я бронь: для себя` или `бронь: для другого`.

## 11. Чего в админ-панели пока нет

В README упоминаются функции, которых **в коде бота нет**:

- кнопка **«Скачать CSV»** в меню;
- команда **`/setstatus`** для смены статуса из Telegram.

Статусы и выгрузка — через **Google Sheets** вручную или через таблицу в браузере.

---

## 12. Частые проблемы

| Проблема | Решение |
|----------|---------|
| Не приходят уведомления о чеках | Проверить `ADMIN_CHAT_ID`, написать боту `/start` с админ-аккаунта |
| Кнопка «Подтвердить» не работает | Статус уже не `PAYMENT_SENT` — смотреть таблицу |
| Картинки не отправляются | URL в настройках должен быть прямым HTTPS |
| Рассылка: много ошибок | Пользователи не нажимали `/start` или заблокировали бота |
| Настройка не сохранилась | Лист «Настройки» должен существовать в таблице |
| Админ видит меню участника | Отправить именно **`/start`** или **`/admin`** — тогда откроется админ-меню |

## 13. Краткая шпаргалка

```
/start или /admin     → админ-меню
/cancel               → отмена рассылки / ввода настройки
/diag                 → проверка LLM

Чек от участника      → ✅ Подтвердить оплату
Статистика            → 📊 Статистика
Массовое сообщение    → 📢 Рассылка → аудитория → текст
Тексты/цены/фото      → ⚙ Настройки → параметр → значение
```
