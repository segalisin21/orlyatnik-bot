# Лист «Настройки» (Орлятник 21+): переменные и импорт

Формат в Google Таблице: **колонка A — ключ** (`key`), **колонка B — значение** (`value`). Первая строка может быть заголовком `key` / `value` (бот читает пары построчно; пустые строки и пустое значение в B игнорируются при слиянии с дефолтами из кода).

## Быстрый импорт

1. Откройте файл [`settings-orlyatnik-defaults.json`](settings-orlyatnik-defaults.json) в этом каталоге.
2. В Google Sheets: **Файл → Импорт → Загрузка** или вставьте ключи вручную из JSON (каждый объект — одна строка таблицы: `key` и `value`).
3. Либо сгенерируйте CSV локально после `npm install` в корне репозитория:
   ```bash
   npx ts-node scripts/export-sheet-defaults.ts
   ```
   Появятся `docs/google-sheet-nastroiki-orlyatnik-defaults.csv` и `docs/google-sheet-nastroiki-pizhamnik-defaults.csv` (UTF-8 с BOM для Excel).

## Смысл ключей

| Ключ | Назначение |
|------|------------|
| `REGISTRATION_CLOSED` | `false` / `true` / `да` — закрыта ли регистрация |
| `DATES` | Кратко даты заезда/выезда (для LLM и справок) |
| `NEXT_SHIFT_TEXT` | Ближайшие смены одной строкой |
| `DEFAULT_SHIFT` | Смена по умолчанию при выборе мероприятия |
| `AVAILABLE_SHIFTS` | Список смен **через запятую** — кнопки и текст «Доступные смены» |
| `START_MESSAGE` | Текст после нажатия «Орлятник 21+» |
| `PROGRAM_TEXT` | Текст расписания после обложки «Узнать программу» |

| `CONDITIONS_PRICE_PHOTO` | Инфографика: цены (если задана — вместо длинного `CONDITIONS_TEXT`) |

| `CONDITIONS_TEXT` | Fallback-текст «Условия и стоимость», если нет инфографик |
| `REVIEWS_BUTTON_LABEL` | Подпись кнопки отзывов в главном меню |

| `MANAGER_KRISTINA_URL` | Личка главного организатора |
| `PRICE` | Справочный максимум цены в ₽ (для листа; в LLM — динамика по датам) |
| `DEPOSIT` | Задаток в ₽ (**8000** для Орлятника) |
| `PAYMENT_SBER` | Реквизиты для перевода |
| `PAYMENT_INSTRUCTION` | Текст при выборе «Оплатить переводом» (если задан в листе; иначе подставляется дефолт из скрипта экспорта) |
| `LOCATION` | Локация для ответов и базы знаний |
| `WHAT_INCLUDED` | Что входит (кратко) |
| `WHAT_TO_TAKE` | Что взять с собой |
| `OBJECTION_*` | Ответы на типовые возражения |

| `AFTER_PAYMENT_INSTRUCTION` | Напоминание после выбора оплаты (чек и т.д.) |
| `FIELD_PROMPT_fio` … `FIELD_PROMPT_shift` | Тексты вопросов анкеты по шагам |
| `FIELD_PROMPT_comment` | Вопрос про особенности/аллергии |
| `SECOND_BOOKING_FINAL_BTN` | Подпись кнопки «ещё путёвка» под финальным «Ты в списке» (крон / после подтверждения оплаты) |

## Лист «Участники» / «Пижамник» — заголовки строки 1 (A1:V1)

На листах **«Участники»** и **«Пижамник»** в **первой строке** должны быть такие заголовки (порядок важен; бот пишет диапазон **A:V**):

| Колонка | Заголовок (рекомендуется) | Поле в боте |
|---------|---------------------------|-------------|
| A | user_id | Telegram ID |
| B | username | @username |
| C | chat_id | chat_id |
| D | status | NEW, INFO, FORM_FILLING, … |
| E | fio | ФИО |
| F | city | город |
| G | dob | дата рождения |
| H | companions | с кем едет |
| I | phone | телефон |
| J | comment | особенности |
| K | shift | смена |
| L | payment_proof_file_id | file_id чека |
| M | final_sent_at | финальное сообщение отправлено |
| N | updated_at | обновлено |
| O | created_at | создано |
| P | last_reminder_at | последнее напоминание |
| Q | consent_at | согласие ОПД (ISO) |
| R | yookassa_payment_id | id платежа |
| S | event | orlyatnik / pizhamnik |
| T | Согласие | «Да» если consent_at заполнен |
| U | booking_scope | self / other (вторая путёвка) |
| V | booking_ref | например 123456789-2 |

**Важно:** колонка **S** — это **мероприятие (`event`)**, не «сдвиг» данных. У статуса `NEW` часто заполнены только A–D; анкета (E–L) и event (S) появятся позже.

**Ручная уборка:** удалите лишние дубли с одним `user_id` и статусом `NEW` (оставьте одну активную строку или последнюю подтверждённую).

## Лист «Участники» и вторая путёвка

На один Telegram `user_id` может быть **несколько строк** в «Участники» (например, две подтверждённые путёвки). Бот выбирает «активную» строку: незавершённая бронь (нижняя среди таких), иначе — последняя по строке таблицы среди подтверждённых.

Повторная бронь на **Орлятник 21+**: после подтверждённой путёвки пользователь выбирает **для себя** (подтягиваются ФИО, контакты и т.д. с последней подтверждённой брони, смену выбирает заново) или **для другого** (новая анкета и согласие). Добавляется **новая** строка; предыдущая `CONFIRMED` не меняется; в «Логи» — `second_booking_append`. Вторая путёвка на Пижамник — через менеджера.

## Smoke после деплоя

1. Пять быстрых `/start` с одного аккаунта — не более одной новой строки `NEW` на `user_id`.
2. Текст + кнопка (например `book_place`) — в «Логи» строки `text` и `callback`.
3. Пройти анкету — поля E–L; выбрать мероприятие — колонка S = `event`.
4. В логах Railway при сбое API — `getSheetRows failed` / `appendLog failed`, без повторных append.

## Пижамник

Отдельный лист **«Настройки Пижамник»** — те же два столбца. Дефолты: после запуска скрипта экспорта — файл `google-sheet-nastroiki-pizhamnik-defaults.csv`, либо см. `kbPizhamnik` в [`src/config.ts`](../src/config.ts).

## Примечание

Приветствие до выбора мероприятия (`ROOT_EVENT_CHOICE_MESSAGE`) и флаг `FEATURE_PIZHAMNIK_UI_ENABLED` **в таблицу не выгружаются** — они только в [`src/config.ts`](../src/config.ts).
